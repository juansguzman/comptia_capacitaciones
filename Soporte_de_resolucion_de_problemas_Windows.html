Resolución de Problemas de Red en Windows

Escenario Común

Usuario reporta que su sistema Windows no puede conectarse al servidor de archivos, impidiendo la transferencia de archivos.

Caso remoto: El usuario está en una conferencia y no puede llevar su laptop al departamento de TI.

Habilidades Clave

  • Diagnóstico de conexiones locales y remotas
  • Uso de herramientas integradas de Windows
  • Comunicación efectiva con usuarios no técnicos
Diagrama: Conexión de red fallida

Problemas de conectividad requieren diagnóstico sistemático

Resolución de Problemas de Configuración IP

Estados de Conexión

  • Desconectado: Verificar cable o configuración inalámbrica
  • Conectividad limitada: Sin servidor DHCP disponible (usa dirección APIPA 169.254.x.y)
  • Sin acceso a Internet: Configuración IP válida localmente pero sin conexión externa

Nota: Windows 10 versión 1511 y anteriores usan www.msftncsi.com para el indicador de estado de conectividad (NCSI).

Captura: Estados de conexión de red

Diferentes estados indican distintos problemas de red

Uso del Comando ipconfig

Información Básica

Sin parámetros, muestra dirección IP, máscara de subred y puerta de enlace predeterminada.

Parámetro /all

  • Muestra configuración detallada
  • Incluye servidores DHCP y DNS
  • Dirección MAC y estado NetBIOS

Preguntas clave: ¿Configuración estática o DHCP? ¿Parámetros correctos para la red local?

Ejemplo de salida de ipconfig /all

Gestión de Direcciones IP con ipconfig

Renovación de Dirección IP

Si una concesión DHCP falta o es incorrecta, se puede solicitar una nueva:

Liberar IP

ipconfig /release NombreAdaptador

Elimina la dirección IP actual del adaptador

Renovar IP

ipconfig /renew NombreAdaptador

Solicita una nueva concesión DHCP

Resolución DNS

Mostrar caché DNS

ipconfig /displaydns

Muestra nombres de host resueltos recientemente

Limpiar caché DNS

ipconfig /flushdns

Elimina registros DNS en caché que puedan estar obsoletos

Comandos Esenciales para Redes

hostname

Muestra el nombre configurado en la máquina local

Uso: Identificación del equipo en la red

Restablecimiento de Red

Opción en Configuración > Red e Internet > Estado

Advertencia: Borra configuraciones personalizadas (VPNs, etc.)

Nota Importante

No reinicie servidores sin considerar el impacto en otros usuarios

Solo cuando: El problema es generalizado

Diagnóstico con el Comando ping

Procedimiento Básico

  1. ping 127.0.0.1 - Verifica que TCP/IP está instalado
  2. ping IP_local - Confirma dirección IP correcta
  3. ping puerta_enlace - Prueba comunicación local
  4. ping host_remoto - Verifica comunicación externa

El tiempo de ida y vuelta (ms) ayuda a diagnosticar problemas de latencia.

Secuencia de prueba de conectividad con ping

Interpretación de Errores con ping

Mensaje de Error Significado Causas Probables
Destino inalcanzable No hay respuesta a sondas ARP Host desconectado, IP duplicada o máscara incorrecta
Inalcanzable desde puerta de enlace El router no tiene información de enrutamiento Configuración incorrecta del router o red destino
Sin respuesta (Tiempo de espera agotado) No se recibió respuesta Host remoto apagado o configurado para no responder

Modo continuo: ping -t envía paquetes indefinidamente (CTRL+C para detener).

DNS: Se puede usar con nombres (ping ejemplo.com) o FQDN, pero requiere DNS funcionando.

Rastreo de Rutas con tracert

Función Principal

Traza la ruta que sigue un paquete para llegar a su destino, mostrando cada salto (router) en el camino.

Características Clave

  • Muestra tiempos de respuesta por salto
  • Identifica cuellos de botella en la red
  • Útil para problemas de enrutamiento

Alternativa: pathping combina tracert con ping para medir latencia y pérdida de paquetes.

Resolución de Problemas de DNS

Comando nslookup

Herramienta para consultar servidores DNS y resolver problemas de nombres.

Sintaxis Básica

nslookup -Opción Host Servidor

Ejemplo: nslookup -type=mx comptia.org 8.8.8.8
Consulta registros MX (correo) para comptia.org usando DNS de Google.

Comparación de Servidores

Al consultar diferentes servidores DNS, se pueden identificar problemas de configuración.

Consulta de registros MX con nslookup

Monitoreo de Puertos con netstat

Funcionalidad Principal

Muestra puertos abiertos y conexiones activas en el host local.

Parámetros Clave

  • -a: Incluye puertos UDP
  • -b: Muestra el proceso asociado (requiere permisos de admin)
  • -n: Muestra números en lugar de nombres (más rápido)
  • -e/-s: Estadísticas de Ethernet y protocolos

Uso en troubleshooting: Verificar si puertos de servicios esenciales (compartición de archivos, correo) están abiertos.

Un representante de ventas necesita un controlador actualizado instalado en su laptop para conectarse a una impresora en ubicación remota.

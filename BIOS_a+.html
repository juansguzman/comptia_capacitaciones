4.1.1 BIOS y UEFI - Conceptos Fundamentales

¿Qué es el Firmware?

  • El firmware es código de programa especial almacenado en memoria flash
  • Está estrechamente ligado a las funciones básicas del hardware específico
  • El firmware de la computadora ayuda a inicializar componentes en la placa base
  • Permite cargar el sistema operativo principal

Evolución del Firmware

  • BIOS (Basic Input/Output System): Sistema tradicional usado por muchos años
  • UEFI (Unified Extensible Firmware Interface): Estándar moderno que reemplaza a BIOS

Notas del Presentador:

El firmware es el primer software que se ejecuta cuando enciendes una computadora. Es fundamental entender que sin firmware funcionando correctamente, la computadora no puede arrancar. La transición de BIOS a UEFI representa una evolución significativa en la tecnología de arranque de computadoras.

4.1.1 Ventajas de UEFI sobre BIOS

Características Principales de UEFI:

  • Soporte para CPU de 64 bits: Operación nativa en arquitecturas modernas
  • Interfaz Gráfica de Usuario (GUI): Con soporte para mouse
  • Capacidades de red durante el arranque: Permite diagnósticos y actualizaciones remotas
  • Mejor seguridad de arranque: Incluye Secure Boot para prevenir malware
  • Modo Legacy BIOS: Compatibilidad con sistemas antiguos

Acceso a la Configuración del Firmware

  • Presionar una tecla durante el proceso de arranque
  • Generalmente cuando aparece el logo del fabricante
  • Teclas comunes: Esc, Del, F1, F2, F10, o F12

Notas del Presentador:

UEFI no es solo una actualización de BIOS, es un reemplazo completo con arquitectura moderna. La capacidad de usar mouse y tener una interfaz gráfica hace que sea mucho más fácil de usar. Secure Boot es especialmente importante para la seguridad moderna, ya que previene que software malicioso se ejecute durante el arranque.

4.1.1 Navegación en Programas de Configuración

BIOS Legacy - Navegación por Teclado

  • Usar teclas de flecha para navegar entre opciones
  • Esc: Generalmente regresa a la pantalla anterior
  • Enter: Selecciona o ingresa a una opción
  • F10: Comúnmente guarda cambios y sale
  • Esc sin guardar: Descarta cambios
  • Opción para cargar configuración predeterminada

UEFI - Interfaz Moderna

  • Interfaz gráfica con soporte para mouse
  • Menús más intuitivos y visuales
  • Algunos menús avanzados aún pueden requerir navegación por teclado
  • Indicadores visuales de temperatura y estado del sistema

Notas del Presentador:

Es importante practicar la navegación en ambos tipos de interfaces. En computadoras modernas con arranque rápido, puede ser necesario usar Shift+Reiniciar desde Windows para acceder a las opciones de UEFI. Siempre documenta los cambios realizados en la configuración del firmware.

4.1.3 Opciones de Arranque y Dispositivos

Secuencia de Arranque

La secuencia de arranque o prioridad de dispositivos de arranque determina el orden en que el firmware busca un gestor de arranque.

Dispositivos de Arranque Típicos:

  • Disco Fijo (HDD/SSD):
    • SATA: Conectar disco de arranque al puerto de menor número
    • NVMe/M.2: Ofrece velocidades superiores a SATA
  • Unidad Óptica (CD/DVD/Blu-ray): Para instalación o reparación desde medios ópticos
  • USB: Común para instalaciones de SO y utilidades de recuperación
  • Red/PXE (Preboot Execution Environment): Arranque a través del adaptador de red

Notas del Presentador:

La configuración correcta del orden de arranque es crucial para la instalación del sistema operativo y la resolución de problemas. PXE es especialmente útil en entornos empresariales para despliegues masivos. Siempre verifica el orden de arranque cuando un sistema no arranca correctamente.

4.1.4 Permisos USB

Control de Seguridad USB

  • El firmware del sistema permite habilitar o deshabilitar controladores y adaptadores
  • Los puertos USB pueden representar riesgos de seguridad
  • Control granular de permisos USB:
    • Deshabilitar puertos individuales
    • Deshabilitar todos los puertos USB
    • Configurar políticas de dispositivos específicos

Opciones de Configuración en UEFI:

  • USB Interface: Bloquear/Desbloquear
  • External Ports: Control de puertos externos
  • Bluetooth: Habilitar/Deshabilitar
  • Camera: Control de cámara integrada
  • Card Reader: Lector de tarjetas

Notas del Presentador:

La seguridad USB es crítica en entornos corporativos donde la fuga de datos es una preocupación. Deshabilitar puertos USB puede prevenir tanto el robo de datos como la introducción de malware. Sin embargo, esto debe equilibrarse con las necesidades operativas de los usuarios.

4.1.5 Consideraciones de Ventiladores

Mantenimiento y Configuración

  • Limpieza regular: Prevenir acumulación de polvo que obstruye el flujo de aire
  • Flujo de aire balanceado: Igual cantidad de aire entrando y saliendo
  • Posicionamiento correcto: Basado en termodinámica
  • Evitar configuraciones que empujen aire caliente hacia abajo

Modos de Control de Ventiladores:

  • Balanced (Balanceado): Configuración estándar para uso general
  • Cool (Frío): Ventiladores a mayor velocidad para máximo enfriamiento
  • Quiet (Silencioso): Reduce velocidad, permite temperaturas más altas
  • Fanless (Sin ventilador): Desactiva ventiladores, usa enfriamiento pasivo
  • Custom (Personalizado): Configuraciones personalizadas de velocidad

Notas del Presentador:

El control adecuado de ventiladores es esencial para el rendimiento y longevidad del sistema. El ciclo de trabajo (duty cycle) controla la frecuencia de pulsos de energía al ventilador. Existen muchas utilidades de terceros que pueden acceder a estas configuraciones desde el SO.

4.1.5 Monitoreo de Temperatura

Métodos de Monitoreo

1. Monitoreo Manual:

  • Reiniciar la computadora y entrar a BIOS/UEFI
  • Presionar tecla específica durante el arranque (F2, Delete, F12, o ESC)
  • Buscar entrada relacionada con monitoreo o sensores
  • Ver lecturas en tiempo real incluyendo temperatura del CPU

2. Aplicaciones de Terceros:

  • Monitorean temperaturas a través de sensores
  • Recolectan datos continuamente
  • Muchos sensores tienen umbrales preestablecidos
  • Generan alertas cuando se exceden límites seguros

Notas del Presentador:

El monitoreo de temperatura es crítico para prevenir daños por sobrecalentamiento. Las aplicaciones de terceros como HWMonitor, Core Temp, o SpeedFan proporcionan monitoreo en tiempo real sin necesidad de reiniciar. Los umbrales típicos de temperatura para CPUs están entre 60-80°C bajo carga.

4.1.6 Contraseñas de Arranque y Secure Boot

Tipos de Contraseñas de Arranque

  • Contraseña de Supervisor/Administrador/Setup/BIOS:
    • Restringe acceso al programa de configuración BIOS/UEFI
    • Protege configuraciones del sistema
  • Contraseña de Usuario/Sistema:
    • Bloquea todo el sistema hasta autenticación
    • Previene cualquier acción hasta que el firmware inicialice

Secure Boot

  • Característica UEFI que protege contra malware
  • Asegura que solo se usen bootloaders firmados digitalmente
  • Verifica integridad del bootloader contra claves criptográficas precargadas
  • Si el bootloader está modificado o sin firmar, no se ejecutará

Notas del Presentador:

Las contraseñas de arranque son una capa adicional de seguridad pero pueden ser problemáticas si se olvidan. Secure Boot es esencial para sistemas modernos y es requerido por Windows 11. Las claves de Microsoft, Fedora, openSUSE y Ubuntu vienen precargadas.

4.1.8 Módulos de Plataforma Confiable (TPM)

¿Qué es TPM?

  • Trusted Platform Module: Hardware que almacena de forma segura certificados digitales, claves criptográficas y contraseñas hasheadas
  • Cada chip TPM tiene una clave de respaldo única e inmutable
  • Establece una raíz de confianza

Funciones del TPM

  • Durante el arranque, compara hashes de datos clave del sistema
  • Verifica que firmware, bootloader y kernel del SO no hayan sido alterados
  • Almacena claves en hardware resistente a manipulación
  • Aísla claves del SO y aplicaciones
  • Usado por programas como Windows BitLocker

Notas del Presentador:

TPM es requerido para Windows 11 y proporciona seguridad superior al almacenar claves en hardware dedicado. La diferencia entre encriptación y hashing es importante: la encriptación es reversible con la clave correcta, mientras que el hashing genera un código único pero no puede revertirse.

4.1.8 Módulo de Seguridad de Hardware (HSM)

Alternativas al TPM

  • Una unidad USB extraíble puede almacenar claves criptográficas
  • Útil cuando:
    • La computadora no soporta TPM
    • Como mecanismo de recuperación si el TPM está dañado
    • Si un disco necesita moverse a otra computadora

Características del HSM

  • Hardware Security Module: USB seguro para almacenar material criptográfico
  • "Seguro" significa que requiere autenticación:
    • Contraseña
    • PIN (Número de Identificación Personal)
    • Huella digital

Notas del Presentador:

Los HSM proporcionan portabilidad y flexibilidad que el TPM integrado no puede ofrecer. Son especialmente útiles en entornos donde los discos duros necesitan moverse entre sistemas o para recuperación de emergencia. La autenticación multifactor añade una capa adicional de seguridad.

4.2.1 Solución de Problemas de Energía

Proceso de Encendido

  • PSU convierte voltaje AC (VAC) a voltaje DC (VDC)
  • PSU suministra 12V inmediatamente - ventiladores y discos duros giran
  • Prueba suministros de 5V y 3.3V
  • Una vez estables, envía señal power good al procesador

Diagnóstico de "Sin Energía"

  1. Verificar otros equipos: Descartar falla del circuito o apagón
  2. Probar el tomacorriente: Conectar dispositivo conocido (lámpara)
  3. Verificar conexiones PSU: Todos los interruptores en "encendido"
  4. Probar otro cable de energía: Verificar fusible con multímetro
  5. Desconectar dispositivos extra: PSU puede estar subdimensionada
  6. Probar la PSU: Usar multímetro o probador de fuente

Notas del Presentador:

¡ADVERTENCIA! Las fuentes de alimentación de PC NO son reparables por el usuario. Nunca quites la cubierta de una fuente de alimentación. Si sospechas de la fuente, no la dejes encendida más tiempo del necesario. Observa señales externas como humo o fuego.

4.2.3 Solución de Problemas POST

Power-On Self-Test (POST)

  • POST: Programa diagnóstico en el firmware del sistema
  • Verifica componentes de hardware requeridos para arrancar
  • En computadoras modernas ocurre muy rápidamente

Síntoma: Pantalla en Blanco con Energía Presente

  1. Preguntar qué ha cambiado: ¿Actualización de firmware reciente?
  2. Verificar cables y conexiones: Especialmente después de mantenimiento
  3. Verificar interfaces y dispositivos defectuosos: Quitar uno por uno
  4. Verificar la PSU: Puede no estar enviando señal power good
  5. Verificar CPU o firmware defectuoso: Reemplazar si es posible

Notas del Presentador:

Los códigos de beep son cruciales para diagnóstico cuando no hay pantalla. Algunos motherboards tienen jumpers para configuración que pueden prevenir el arranque si están mal configurados. Siempre verifica los jumpers después del servicio.

4.2.3 Códigos de Beep POST

Códigos de Beep Estándar IBM PC

Código Significado
1 beep corto POST normal - sistema OK
2 beeps cortos Error POST - código mostrado en pantalla
Sin beep PSU, motherboard o altavoz defectuoso
Beep continuo Problema con memoria o controlador
Beeps cortos repetidos Falla PSU o problema motherboard
1 largo, 2-3 cortos Error de adaptador de video

Notas del Presentador:

Los códigos de beep varían según el fabricante del BIOS. Siempre consulta la documentación del fabricante. Algunas PCs no arrancarán si una tecla está presionada - verifica que nada esté sobre el teclado.

4.2.5 Solución de Problemas de Arranque

Proceso de Arranque Post-POST

  • Sistema busca dispositivos de arranque en orden especificado
  • Si no encuentra el primero, intenta el siguiente
  • Error si no encuentra dispositivo arrancable

Si el Disco Fijo No es Detectado:

  • Verificación de Energía:
    • Buscar LED de actividad
    • Escuchar si el disco gira
    • Verificar conectores de energía
  • Conexiones de Datos: Inspeccionar cables por daños
  • Configuración UEFI/BIOS:
    • Confirmar que la unidad está habilitada

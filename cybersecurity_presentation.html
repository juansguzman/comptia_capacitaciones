Lo Opuesto de la Tríada CIA

La Tríada DAD

Mientras que la Tríada CIA (Confidencialidad, Integridad, Disponibilidad) representa los objetivos de seguridad, la Tríada DAD representa las amenazas que debemos combatir:

  • Disclosure (Divulgación)
  • Alteration (Alteración)
  • Destruction (Destrucción)

Disclosure (Divulgación)

Es análogo a una violación de la confidencialidad. Se refiere al acceso no autorizado y exposición de información.

Ejemplo: Un atacante obtiene acceso a una base de datos de clientes y expone información personal como números de tarjetas de crédito, direcciones y contraseñas.

Impacto: Pérdida de confianza, multas regulatorias, robo de identidad.

Alteration (Alteración)

Esta amenaza corresponde a una pérdida de integridad, donde se realizan cambios no autorizados a los datos.

Ejemplo: Un empleado descontento modifica los registros financieros de la empresa para ocultar fraude o crear transacciones falsas.

Consecuencias: Decisiones empresariales incorrectas, pérdidas financieras, problemas legales.

Destruction (Destrucción)

Representa un ataque directo a la disponibilidad. La destrucción implica la eliminación o corrupción de datos o daño físico al hardware, haciendo la información irrecuperable.

Ejemplo: Ransomware que cifra todos los archivos de una organización y elimina las copias de seguridad.

Impacto: Interrupción del negocio, pérdida permanente de datos críticos, costos de recuperación.

El Marco AAA

AAA es un marco fundamental en ciberseguridad que garantiza el control de acceso seguro a los recursos:

  • Authentication (Autenticación): ¿Quién eres?
  • Authorization (Autorización): ¿Qué puedes hacer?
  • Accounting (Contabilidad/Auditoría): ¿Qué hiciste?

Authentication (Autenticación)

Es el proceso de verificar la identidad de un usuario, dispositivo u otra entidad en un sistema informático, típicamente como prerrequisito para otorgar acceso a recursos en un sistema.

• Viene después de la identificación

Proceso típico:
1. Identificación: "Soy usuario123"
2. Autenticación: "Mi contraseña es ********"
3. Verificación: El sistema confirma que la contraseña coincide

Factores de Autenticación

Algo que sabes

Verificar identidad basándose en conocimiento confidencial: contraseñas, PIN, respuestas a preguntas secretas.

Algo que tienes

Elementos en tu posesión: tokens de seguridad, tarjetas inteligentes, teléfono móvil (para OTPs).

Algo que eres

Biometría - características físicas únicas: huellas dactilares, reconocimiento facial, escaneos de iris, patrones de voz.

Donde estás

Ubicación del usuario determinada por IP, GPS u otros métodos de geolocalización.

Algo que haces

Biometría conductual: dinámica de tecleo, patrones de uso del mouse.

Autenticación de Múltiples Factores (MFA)

Proceso de seguridad que requiere más de un método de autenticación de categorías independientes de credenciales para verificar la identidad del usuario.

MFA combina dos o más credenciales independientes: lo que el usuario sabe (contraseña), lo que el usuario tiene (token de seguridad), y lo que el usuario es (verificación biométrica).

Ejemplo común: Iniciar sesión en tu banco requiere tu contraseña + un código SMS enviado a tu teléfono

Autenticando Personas

Se puede hacer usando:

  • Biometría: Utilizar características físicas (ej., huellas dactilares, reconocimiento facial, escaneos de retina) únicas de un individuo.
  • Autenticación basada en conocimiento: Requerir información que solo el usuario debería saber (ej., contraseñas, PINs, preguntas de seguridad).
  • Autenticación de múltiples factores (MFA): Combinar algo que el usuario sabe (contraseña) con algo que tiene (teléfono o token) o es (verificación biométrica).

Autenticando Sistemas

Se puede hacer usando:

  • Certificados y Claves: Usar certificados digitales y claves criptográficas para establecer confianza entre máquinas.
  • Lista de IP permitidas: Permitir solo sistemas con ciertas direcciones IP acceder a un servicio o red.
  • Filtrado de direcciones MAC: Restringir acceso a una red a dispositivos con direcciones MAC específicas.

Authorization (Autorización)

La autorización determina qué puede hacer ese usuario estableciendo sus derechos y privilegios.

Se puede hacer usando:

  • Permisos y Privilegios: Implica otorgar permisos para acceder a recursos específicos o datos. Los permisos definen las acciones permitidas, como leer, escribir, ejecutar, eliminar o modificar.
  • Control de Acceso: La autorización se aplica a través de mecanismos de control de acceso como las Listas de Control de Acceso (ACLs).
  • Modelos de Autorización: Como Control de Acceso Obligatorio (MAC) o Control de Acceso Discrecional (DAC).

Accounting (Contabilidad/Auditoría)

Se refiere al seguimiento de las actividades del usuario y el uso de recursos dentro de un sistema.

Garantiza que los usuarios no solo estén autenticados y autorizados, sino también responsables de sus acciones mientras acceden y usan recursos del sistema.

Se puede hacer usando:

  • Seguimiento de actividad del usuario: La contabilidad implica recopilar datos sobre actividades del usuario, como tiempos de inicio de sesión, duración de sesiones, recursos accedidos, servicios de red utilizados, cambios del sistema realizados y datos transferidos.
  • Rastros de auditoría y registros: Los sistemas mantienen registros y rastros de auditoría que capturan información detallada sobre varios eventos.

Accountability (Responsabilidad)

Principio que garantiza que las personas o entidades sean responsables de sus acciones dentro de un sistema.

La responsabilidad es crucial porque:

  • Permite el rastreo de acciones hasta usuarios específicos
  • Facilita las investigaciones forenses
  • Disuade el comportamiento malicioso
  • Cumple con requisitos regulatorios

Gap Analysis (Análisis de Brechas)

Es una evaluación metódica que las organizaciones usan para comparar su postura de seguridad actual con un conjunto de estándares, mejores prácticas o requisitos regulatorios para identificar áreas que necesitan mejora.

Implica:

  • Identificación del Estado Actual: Esto implica mapear las medidas de seguridad, políticas y controles existentes.
  • Determinación del Estado Objetivo: El estado objetivo generalmente está definido por estándares de la industria, requisitos regulatorios o los objetivos internos de seguridad de la organización.
  • Análisis de la Brecha: El núcleo del análisis de brechas es identificar las diferencias entre el estado actual y el estado objetivo.

Controles de Seguridad

Los controles de seguridad son salvaguardas o contramedidas empleadas para evitar, detectar, contrarrestar o minimizar riesgos de seguridad a la propiedad física, información, sistemas informáticos u otros activos.

Ayudan a mantener la confidencialidad, integridad y disponibilidad de la información

Se dividen en:

Categorías

  • Técnicos
  • Gerenciales
  • Operacionales
  • Físicos

Tipos

  • Preventivos
  • Disuasorios
  • Detectivos
  • Correctivos
  • Compensatorios
  • Directivos

Controles de Seguridad - Categorías

Controles Técnicos

Los controles de seguridad técnicos, también conocidos como controles de seguridad lógicos, son mecanismos implementados en hardware, software o firmware que automatizan el proceso de prevenir, detectar y responder a amenazas de seguridad.

Incluye:

  • Mecanismos de Control de Acceso
  • Firewalls
  • Sistemas de Detección de Intrusiones (IDS) y Sistemas de Prevención de Intrusiones (IPS)
  • Cifrado
  • Software Antivirus y Antimalware
  • Redes Privadas Virtuales (VPN)

Controles de Seguridad - Categorías

Controles Gerenciales

Los controles de seguridad gerenciales, también conocidos como controles administrativos, son las políticas, procedimientos y pautas que gobiernan el comportamiento de las personas dentro de una organización y la operación de los sistemas de TI.

Incluye:

  • Políticas y Procedimientos de Seguridad
  • Gestión de Riesgos
  • Planes de Respuesta a Incidentes y Recuperación
  • Planificación de Continuidad del Negocio y Recuperación ante Desastres

Controles de Seguridad - Categorías

Controles Operacionales

Los controles de seguridad operacionales son los métodos y procedimientos del día a día que se implementan en una organización para garantizar y mantener la seguridad de su información y activos.

Realizados por personas en la organización.

Incluye:

  • Capacitación de Concientización sobre Seguridad
  • Protección de medios físicos

Controles de Seguridad - Categorías

Controles Físicos

Los controles de seguridad física en seguridad de TI son medidas tomadas para proteger el hardware real y las instalaciones que albergan los sistemas, redes y datos.

Estos controles están diseñados para prevenir el acceso no autorizado, daño e interferencia a los recursos físicos de la organización.

Incluye:

  • Iluminación
  • Señalización
  • Cercas
  • Guardias de seguridad
  • Cámaras

Tipos de Controles de Seguridad

Controles Preventivos

Intentan detener que ocurra un incidente de seguridad.

Ej.: IPS, firewalls, cifrado, controles de acceso

Controles Detectivos

Intentan detectar eventos que resultaron en un incidente de seguridad.

Ej.: IDS, SIEM, videovigilancia, detección de movimiento

Controles Correctivos

Intentan remediar un incidente que ha ocurrido.

Ej.: UPS, restauración de copias de seguridad, procedimientos de respuesta a incidentes

Tipos de Controles de Seguridad (Continuación)

Controles Disuasorios

Intentan desalentar una amenaza.

Ej.: Perros guardianes, cámaras, alambre de púas

Controles Directivos

Proporcionan direcciones sobre cómo operar los sistemas.

Ej.: Políticas, procedimientos

Controles Compensatorios

Proporcionan controles alternativos cuando el control principal puede no ser suficiente.

Ej.: Segregación de funciones

Shadow IT (TI en la Sombra)

Se refiere a sistemas y soluciones de tecnología de la información construidos y utilizados dentro de las organizaciones sin aprobación organizacional explícita.

Incluye software, hardware y servicios en la nube adquiridos y gestionados fuera del ámbito del departamento oficial de TI.

Los riesgos asociados con Shadow IT provienen de la falta de supervisión y control.

Los sistemas y aplicaciones que no son evaluados por el departamento de TI pueden no cumplir con las políticas o estándares de seguridad organizacional.

Esto puede llevar a violaciones de datos, incumplimiento de regulaciones y vulnerabilidades potenciales en la red.

Ejemplos comunes: Empleados usando servicios de almacenamiento en la nube no autorizados para compartir archivos de la empresa, instalando software personal en dispositivos de trabajo, o usando aplicaciones de mensajería no sancionadas para comunicación.

Vectores de Amenaza y Superficies de Ataque

Basados en Mensajes:

  • Email: Un medio popular para entregar contenido o enlaces maliciosos. Los intentos de phishing, malware, ransomware y spam a menudo usan este vector.
  • SMS: Los mensajes de texto móviles pueden contener enlaces de phishing (Smishing) o contenido malicioso dirigido a smartphones.
  • Mensajería Instantánea (IM): Los servicios de mensajería en tiempo real pueden ser explotados para entregar malware o contenido de phishing.

Otros Vectores:

  • Basado en Imágenes: Las cargas maliciosas pueden estar incrustadas en imágenes que, al ser vistas, pueden explotar vulnerabilidades.
  • Basado en Archivos: El software malicioso puede estar incrustado dentro de archivos que, al abrirse o ejecutarse, pueden llevar a comprometer el sistema.

Vectores de Amenaza: Voz y Dispositivos

Llamada de Voz

Vishing (phishing basado en voz) involucra a criminales usando llamadas telefónicas para engañar a las víctimas para que divulguen información personal o sigan instrucciones maliciosas.

Ejemplo: "Hola, soy del departamento de TI. Necesitamos su contraseña para resolver un problema urgente."

Dispositivo Extraíble

Dispositivos como USBs pueden usarse para introducir malware o explotar vulnerabilidades de software cuando se conectan a un sistema.

Software Vulnerable:

  • Basado en Cliente: Software que requiere instalación en el sistema de un usuario puede ser objetivo de vulnerabilidades.
  • Sin Agente: Software que se ejecuta sin instalaciones o agentes, haciéndolos más difíciles de monitorear y potencialmente vulnerables.

Vectores de Amenaza: Redes y Sistemas

Sistemas y Aplicaciones No Soportados

El software desactualizado que ya no recibe actualizaciones de seguridad puede ser un riesgo significativo.

Redes Inseguras:

  • Inalámbricas: Las redes Wi-Fi no seguras pueden ser interceptadas o explotadas.
  • Cableadas: El acceso físico a redes cableadas puede llevar a intrusiones.
  • Bluetooth: Las vulnerabilidades en Bluetooth pueden ser explotadas para espiar o controlar dispositivos.

Puertos de Servicio Abiertos

Los puertos abiertos no seguros pueden permitir acceso no autorizado o ataques a servicios que se ejecutan en esos puertos.

Credenciales Predeterminadas

Dispositivos o sistemas con contraseñas predeterminadas sin cambiar pueden ser fácilmente accedidos por atacantes.

Ejemplos comunes: admin/admin, admin/password, root/toor, admin/1234

Cadena de Suministro

Proveedores de Servicios Gestionados (MSPs)

Si se comprometen, pueden proporcionar acceso a la infraestructura de sus clientes.

Vendedores

Sus sistemas, si son violados, pueden actuar como una puerta de entrada a la infraestructura de una organización.

Proveedores

Un compromiso en la seguridad de un proveedor puede tener efectos en cadena en sus clientes.

Caso Real: El ataque SolarWinds afectó a miles de organizaciones a través de una actualización de software comprometida

Vulnerabilidades

Una vulnerabilidad se refiere a una debilidad en un sistema que puede ser explotada por un actor de amenaza, como un hacker, para obtener acceso no autorizado o realizar acciones no autorizadas en un sistema informático.

Tipos principales de vulnerabilidades:

  • Inyección de memoria
  • Desbordamiento de búfer
  • Condiciones de carrera
  • Vulnerabilidades del sistema operativo
  • Inyección SQL
  • Cross-Site Scripting (XSS)
  • Vulnerabilidades de hardware
  • Vulnerabilidades de virtualización
  • Vulnerabilidades específicas de la nube

Inyección de Memoria

Insertar código malicioso en la memoria de un programa. El atacante aprovecha vulnerabilidades que le permiten ejecutar código arbitrario.

Las técnicas comunes incluyen:

  • Inyección de código
  • Desbordamientos de búfer
  • Inyecciones de DLL

Se corrige siguiendo prácticas de codificación segura como la validación de entrada.

Prevención: Siempre validar y sanear la entrada del usuario antes de procesarla

Desbordamiento de Búfer

Un desbordamiento de búfer ocurre cuando los datos que se pretenden almacenar en un búfer, que es un bloque contiguo de memoria de computadora, exceden la capacidad de almacenamiento del búfer. Esto resulta en que las ubicaciones de memoria adyacentes sean sobrescritas.

Ejemplo de desbordamiento de búfer basado en pila:

Considera un programa que pide el nombre de un usuario y lo almacena en un búfer que puede contener 10 caracteres. Si el usuario ingresa un nombre de 12 caracteres, los 2 caracteres adicionales podrían sobrescribir la memoria adyacente. Si esta memoria adyacente está controlando el flujo de ejecución del programa (por ejemplo, dirección de retorno para una función), un atacante puede manipular esto para ejecutar código arbitrario.

Generalmente detectado durante un escaneo de vulnerabilidades.

Se corrige siguiendo buenas prácticas de codificación como verificar la longitud de los datos antes de escribirlos en un búfer.

Condiciones de Carrera

Vulnerabilidad que ocurre cuando el tiempo de las acciones afecta el estado y el resultado de un sistema.

El peligro surge cuando el éxito de una operación de seguridad depende del tiempo de ciertos eventos, y una entidad maliciosa puede influir en este tiempo.

Vulnerabilidades de Tiempo de Verificación a Tiempo de Uso (TOCTOU):

Estas vulnerabilidades ocurren cuando un programa verifica el estado de un recurso y luego lo usa después de un retraso, durante el cual el estado puede haber cambiado. Si un atacante puede cambiar el recurso entre los pasos de "verificación" y "uso", puede inducir un comportamiento no autorizado.

Error TOCTOU clásico: Se encuentra en operaciones de acceso a archivos. Si un programa verifica la existencia de un archivo y luego lo abre, un atacante podría intercambiar el archivo con un enlace simbólico a un archivo sensible después de la verificación pero antes del uso. El programa, que ahora abre el enlace simbólico, puede inadvertidamente leer, escribir o eliminar información sensible.

Ejemplo de Condición de Carrera

Ejecución normal de un archivo en Unix:

if (access("file", W_OK) != 0) {
  exit(1);
}
fd = open("file", O_WRONLY);
write(fd, buffer, sizeof(buffer));

Ataque:

if (access("file", W_OK) != 0) {
  exit(1);
}
// El atacante crea un enlace simbólico aquí
symlink("/etc/passwd", "file");
fd = open("file", O_WRONLY);
write(fd, buffer, sizeof(buffer));

¡El atacante logra escribir en /etc/passwd en lugar del archivo original!

Vulnerabilidades Basadas en el Sistema Operativo (SO)

Debilidades en el SO que pueden ser explotadas para obtener acceso no autorizado, elevar privilegios, etc.

Generalmente se resuelven actualizando el sistema operativo.

Muchas empresas todavía usan SO que ya no están soportados como:

  • Windows XP
  • Windows 7

Esto lleva a que no haya actualizaciones disponibles para los sistemas operativos.

Riesgo: Las vulnerabilidades no parcheadas pueden ser explotadas años después de su descubrimiento

Inyección de Lenguaje de Consulta Estructurado (SQLi)

Los atacantes insertan código SQL malicioso en campos de entrada para ejecutar consultas SQL no autorizadas.

Ejemplo y Demo:

https://www.codingame.com/playgrounds/154/sql-injection-demo/sql-injection

Ejemplo de Ataque:
Login: admin' OR '1'='1
Password: cualquier cosa

Esto hace que la consulta SQL sea:
SELECT * FROM users WHERE username = 'admin' OR '1'='1' AND password = 'cualquier cosa'

Puede corregirse usando prácticas de codificación segura como la validación de entrada.

Cross-Site Scripting (XSS)

Vulnerabilidad de seguridad típicamente encontrada en aplicaciones web. XSS permite a los atacantes inyectar scripts del lado del cliente en páginas web vistas por otros usuarios.

Una vulnerabilidad XSS surge cuando una aplicación web incluye entrada de usuario no validada o no codificada como salida en una página.

Ejemplo y Demo:

https://xss-game.appspot.com/

Ejemplo de ataque XSS:
<script>alert('XSS')</script>
Si este script no se filtra, se ejecutará en el navegador de la víctima

Puede corregirse usando prácticas de codificación segura como la validación de entrada.

Vulnerabilidades de Hardware

Vulnerabilidades de Firmware

Debilidades en el software de bajo nivel que se ejecuta en dispositivos de hardware.

Hardware al Final de su Vida Útil

Dispositivos que ya no son soportados por los fabricantes, resultando en vulnerabilidades sin parchar.

Hardware Heredado

Hardware más antiguo que puede no ser compatible con las medidas de seguridad actuales.

Ejemplo: Routers con firmware desactualizado que no han sido actualizados en años, switches antiguos sin soporte de cifrado moderno

Vulnerabilidades de Virtualización

Escape de Máquina Virtual (VM)

Un atacante ejecuta código en una VM que le permite salir e interactuar con el sistema host.

Reutilización de Recursos

Los datos sensibles pueden permanecer en los recursos del sistema y ser accedidos por otros procesos.

Riesgo: Si un atacante escapa de una VM, podría acceder a todas las otras VMs en el mismo host, comprometiendo múltiples sistemas a la vez

Vulnerabilidades Específicas de la Nube

La computación en la nube se ha vuelto cada vez más popular, pero también introduce vulnerabilidades únicas que son específicas de su entorno.

Vulnerabilidades específicas de la nube:

  • Violaciones de Datos: Los datos almacenados en servidores en la nube pueden ser objetivo de hackers, exponiendo potencialmente información sensible.
  • Gestión Insuficiente de Identidad, Credenciales y Acceso: Procesos de autenticación débiles, gestión inadecuada de credenciales y controles de acceso insuficientes pueden llevar a acceso no autorizado a recursos en la nube.
  • Interfaces y APIs Inseguras: Los servicios en la nube se acceden a través de interfaces y APIs que, si no están adecuadamente aseguradas, pueden ser explotadas.
  • Vulnerabilidades del Sistema: Las infraestructuras en la nube pueden ser complejas y podrían contener vulnerabilidades del sistema.
  • Secuestro de Cuenta: Un atacante que obtiene acceso a la cuenta en la nube de un usuario puede manipular datos, espiar transacciones y redirigir clientes a sitios ilegítimos.

Vulnerabilidades de la Cadena de Suministro

Se refiere a la compleja red de proveedores involucrados en la producción y distribución de productos y servicios de TI. Esto incluye proveedores de servicios, proveedores de hardware y proveedores de software.

Proveedor de Servicios:

Estas entidades entregan diversos servicios de TI, incluyendo computación en la nube, almacenamiento de datos y servicios de red.

Preocupaciones de Seguridad: Los proveedores de servicios podrían ser un vector para violaciones de seguridad si sus sistemas están comprometidos. Tienen acceso a datos sensibles y a menudo se integran con los sistemas internos de sus clientes.

Proveedores de Hardware y Software

Proveedor de Hardware:

Estos son los fabricantes y distribuidores de dispositivos y componentes de computación física, como servidores, routers y chips.

Preocupaciones de Seguridad: La integridad del hardware es crucial. El hardware puede ser manipulado en cualquier punto de la cadena de suministro, llevando a riesgos como malware integrado o puertas traseras.

Proveedor de Software:

Entidades que desarrollan y distribuyen software, incluyendo sistemas operativos, aplicaciones y firmware.

Preocupaciones de Seguridad: Las vulnerabilidades del software pueden ser explotadas por atacantes para obtener acceso no autorizado o interrumpir servicios. Este riesgo se magnifica si el software es ampliamente utilizado en múltiples organizaciones. Las actualizaciones regulares, la gestión de parches y las auditorías de seguridad son esenciales para mitigar estos riesgos.

Vulnerabilidades Criptográficas

Se refieren a debilidades dentro de los algoritmos criptográficos o su implementación que pueden ser explotadas para violar la seguridad.

Pueden incluir:

  • Debilidades del Algoritmo: Algunos algoritmos criptográficos tienen debilidades inherentes. Por ejemplo, algoritmos más antiguos como DES ya no se consideran seguros porque pueden romperse con suficiente poder computacional.
  • Problemas de Gestión de Claves: Si las claves no se generan, almacenan o manejan de forma segura, pueden comprometerse.
  • Implementación Deficiente: Esto puede incluir errores de programación, como desbordamientos de búfer, que pueden ser explotados para obtener acceso o información no autorizada.
Ejemplo: Usar MD5 para hash de contraseñas - ¡ha sido roto desde 2004! Use SHA-256 o bcrypt en su lugar.

Vulnerabilidades de Mala Configuración

Se refieren a la configuración inadecuada de software, hardware o sistemas de red, lo que puede llevar a debilidades de seguridad.

Incluye:

  • Configuraciones predeterminadas
  • Servicios innecesarios habilitados
  • Controles de seguridad inadecuados
Ejemplos comunes:
• Dejar el puerto 3389 (RDP) abierto a Internet
• Usar certificados SSL autofirmados en producción
• Permisos de archivos demasiado permisivos (777)
• Bases de datos sin contraseña

Vulnerabilidades de Dispositivos Móviles

Se refieren a debilidades que podrían ser potencialmente explotadas por atacantes para obtener acceso no autorizado a un dispositivo o sus datos.

Incluye:

  • Sistemas Operativos Desactualizados: No instalar actualizaciones puede dejar agujeros de seguridad conocidos sin parchar.
  • Conexiones de Red No Seguras: Usar conexiones Wi-Fi o Bluetooth no cifradas puede exponer datos.
  • Acceso Físico: Un dispositivo desbloqueado o no seguro puede ser fácilmente manipulado.
  • Fallas del Sistema: Debilidades inherentes en el sistema operativo o hardware que pueden ser explotadas.
  • Comportamiento del Usuario: Como compartir contraseñas o hacer clic en enlaces de phishing.

Jailbreaking y Sideloading

Jailbreaking

Se refiere al proceso de eliminar las restricciones de software impuestas por el sistema operativo en dispositivos, particularmente dispositivos iOS. Para Android, un proceso similar se conoce como "rooting".

• Generalmente se hace reemplazando el SO en el dispositivo con un SO que te da acceso root.

Side loading

Es el proceso de instalar aplicaciones en un dispositivo móvil desde fuentes distintas a la tienda oficial de aplicaciones (como Google Play Store para Android o Apple App Store para iOS).

Riesgos: Las aplicaciones sideloaded pueden contener malware, no reciben actualizaciones automáticas, y pueden comprometer la seguridad del dispositivo

Otros Signos de Ataques: Bloatware

Se refiere a software no deseado que viene preinstalado en un dispositivo, típicamente por el fabricante, o se incluye en otras instalaciones de software.

No es inherentemente malicioso como el malware, pero el bloatware puede ralentizar los sistemas, ocupar espacio valioso en el disco, y a veces, puede incluir vulnerabilidades que podrían ser explotadas por actores maliciosos.

Características del Bloatware:

  • Aplicaciones preinstaladas
  • Consumo de recursos
  • Difícil de eliminar
  • Riesgos potenciales de seguridad
Ejemplo: Aplicaciones de prueba que vienen con nuevas computadoras, barras de herramientas del navegador no deseadas, software del fabricante que nunca usas

Denegación de Servicio Distribuida (DDoS)

Un intento malicioso de interrumpir el tráfico normal de un servidor, servicio o red objetivo al abrumar el objetivo o su infraestructura circundante con una inundación de tráfico de Internet.

Utilizan múltiples sistemas informáticos comprometidos como fuentes de tráfico de ataque.

Estos sistemas pueden incluir computadoras y otros recursos en red como dispositivos IoT.

Mapa de DDoS en vivo

https://www.netscout.com/ddos-attack-map

Tipos de Ataques DDoS

DDoS Basado en Red

Un perpetrador usa múltiples sistemas comprometidos, a menudo infectados con un troyano, para lanzar un único ataque masivo. Estos sistemas forman una red llamada botnet.

Inundaciones UDP

El atacante abruma puertos aleatorios en el host objetivo con paquetes IP que contienen datagramas UDP. El objetivo es inundar la red con suficientes paquetes UDP para ralentizar o bloquear el sistema objetivo.

Inundaciones SYN

Un ataque de inundación SYN es un tipo de ataque de Denegación de Servicio (DoS) que apunta a la secuencia de conexión TCP, conocida como el protocolo de enlace de tres vías TCP.

Ataques de Amplificación y Mitigación DDoS

Ataques de Amplificación

Estos ataques explotan las características de ciertos protocolos para magnificar la cantidad de tráfico que se envía a un objetivo, causando una denegación de servicio.

• Usa protocolos como DNS o direccionamiento IP

DDoS Reflejado

Caracterizado por su uso de reflexión, lo que significa que el atacante obliga a servidores de terceros a dirigir tráfico a la víctima, a menudo sin el conocimiento del tercero.

• La suplantación de IP es una forma de hacer esto.

Los ataques DDoS pueden mitigarse mediante:

  • Aumentar el ancho de banda
  • Servicios de protección DDoS (Cloudflare) - https://www.cloudflare.com/ddos/
  • Hardware de red con protección DDoS: Algunos hardware de red, como routers y firewalls, vienen con características de protección DDoS incorporadas.

Sistema de Nombres de Dominio (DNS)

DNS es esencialmente la guía telefónica de Internet; traduce nombres de dominio legibles por humanos (como www.example.com) en direcciones IP numéricas que las computadoras usan para conectarse entre sí.

Varias preocupaciones de seguridad y vectores de ataque:

  • Suplantación de DNS (o Envenenamiento de Caché): Este ataque implica corromper el caché DNS con información falsa.
  • Ataques de Amplificación DNS: Son un tipo de ataque DDoS donde el atacante explota servidores DNS de acceso público.
  • Túnel DNS: Implica codificar los datos de otros programas o protocolos en consultas y respuestas DNS.
  • Secuestro de DNS: El atacante desvía consultas a un servidor DNS malicioso.

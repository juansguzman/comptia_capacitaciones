Sistema de computación con dispositivos periféricos

¿Qué son los Dispositivos Externos?

Definición

Los dispositivos externos o periféricos son componentes hardware que permiten la comunicación entre el sistema de computación y el mundo exterior.

Características Principales

  • Interfaz: Proporcionan punto de contacto con usuarios
  • Transducción: Convierten entre dominios físicos y eléctricos
  • Heterogeneidad: Gran variedad de tipos y velocidades

Arquitectura General de Periféricos

Componentes Comunes

  • Transductor: Convierte señales entre dominios
  • Buffer: Almacenamiento temporal de datos
  • Controlador: Lógica de control y protocolo
  • Interfaz: Conexión con bus del sistema

Nota: Cada componente puede implementarse en hardware, software, o combinación de ambos.

Dispositivos de Entrada

Convierten información del mundo físico en datos digitales que el sistema puede procesar.

Ejemplos Principales

  • Teclado: Texto y comandos
  • Ratón: Posición y clics
  • Micrófono: Audio
  • Cámara: Imágenes y video
  • Sensores: Datos ambientales

Clasificación por Velocidad de Transferencia

Importancia de la Velocidad

La velocidad de transferencia determina estrategias de buffering, protocolos de comunicación y arquitectura de controladores.

Desafío: Manejar eficientemente dispositivos con velocidades que difieren por varios órdenes de magnitud.

Comparación de velocidades de transferencia

Dispositivos de Baja Velocidad

Rango: 1-100 bytes/segundo

Características

  • Limitados por humanos: Velocidad de tipeo, movimiento
  • Eventos esporádicos: No transferencia continua
  • Baja latencia requerida: Respuesta inmediata

Ejemplos

  • Teclados (5-10 caracteres/seg típico)
  • Ratones (posición + clics)
  • Sensores ambientales

Dispositivos de Velocidad Media

Rango: 100 bytes - 1MB/segundo

Características

  • Transferencias sustanciales: Bloques de datos
  • Buffering importante: Suavizar variaciones
  • Operación continua: Flujos de datos

Ejemplos

  • Impresoras de inyección (~1-10 KB/s)
  • Escáneres de documentos
  • Audio CD calidad (~176 KB/s)
  • Modems dial-up (~7 KB/s)

Impresora de inyección - Velocidad media

Dispositivos de Alta Velocidad

Rango: >1MB/segundo

Características

  • Saturan buses: Pueden limitar sistema
  • DMA esencial: Transferencia directa a memoria
  • Buffering extensivo: Múltiples niveles
  • Optimización crítica: Impacto en rendimiento

Ejemplos

  • Discos duros (100-200 MB/s)
  • SSD (500-7000 MB/s)
  • Video HD sin comprimir (>100 MB/s)
  • Red Gigabit Ethernet (~125 MB/s)

Categorías por Método de Acceso

Importancia del Método de Acceso

El método de acceso determina cómo se organizan y recuperan los datos, afectando directamente el rendimiento y las estrategias de optimización.

Impacto: Algoritmos de scheduling, cacheing y buffering dependen fundamentalmente del método de acceso.

Cinta magnética - Acceso secuencial

Acceso Secuencial

Características

  • Orden fijo: Datos deben accederse en secuencia
  • Tiempo variable: Depende de posición actual
  • Rewind necesario: Para volver al inicio
  • Optimización específica: Minimizar movimientos

Aplicaciones

  • Backup y archivado
  • Procesamiento batch
  • Streaming de media

Acceso Aleatorio

Características

  • Flexibilidad: Cualquier ubicación accesible
  • Tiempo variable: Depende de distancia a recorrer
  • Seek time: Tiempo de posicionamiento
  • Optimizable: Algoritmos de scheduling

Ventajas

  • Acceso directo a cualquier dato
  • Ideal para bases de datos
  • Soporte para multitasking

Mecanismo interno - Acceso aleatorio con seek

Algoritmos de Scheduling para Acceso Aleatorio

Algoritmo Estrategia Ventajas Desventajas
FIFO Primero en llegar Simple, equitativo No optimiza seeks
SSTF Seek más corto Minimiza tiempo seek Puede causar starvation
SCAN Elevator algorithm Evita starvation Favorece centro del disco
C-SCAN Circular scan Tiempo uniforme Overhead de retorno

Acceso Directo

Características

  • Sin movimiento mecánico: Acceso electrónico puro
  • Tiempo constante: Latencia uniforme
  • Alta velocidad: Sin delays de posicionamiento
  • Desgaste uniforme: Wear leveling

Ventajas

  • Latencia mínima y predecible
  • Sin ruido mecánico
  • Menor consumo energético
  • Mayor durabilidad física

Señales de Comunicación

Importancia Fundamental

Las señales de comunicación forman el "lenguaje" mediante el cual el procesador y los dispositivos periféricos coordinan sus operaciones.

Tipos Principales

  • Control: Coordinan operaciones
  • Estado: Informan condiciones
  • Datos: Transportan información

Sin estas señales: No habría comunicación organizada entre componentes del sistema.

Niveles de Abstracción

Protocolos de Handshaking

Handshaking Asíncrono

  1. Strobe: Transmisor indica datos válidos
  2. Acknowledge: Receptor confirma recepción
  3. Release: Transmisor libera strobe
  4. Ready: Receptor libera acknowledge

Ventaja: Funciona con dispositivos de diferentes velocidades sin sincronización externa.

Registro de Estado del Dispositivo

Organización Típica

  • Bit 0: Ready - Dispositivo listo
  • Bit 1: Busy - Operación en curso
  • Bit 2: Error - Error detectado
  • Bit 3: IRQ - Interrupción pendiente
  • Bits 4-7: Código de error específico

Polling: El sistema puede leer este registro periódicamente para monitorear estado.

Servidores de Archivos e Impresión

Funciones Principales

  • File Share: Disco configurado para acceso de clientes por red
  • Arquitectura Cliente-Servidor: Máquina host como servidor, dispositivos como clientes
  • Protocolos: Propietarios (Windows) o TCP/IP (FTP)

Beneficios

  • Acceso compartido centralizado a recursos
  • Gestión eficiente de impresión empresarial
  • Control de colas de impresión
  • Liberación controlada de trabajos

Servidor de archivos e impresión HP

Nota: Los servidores de impresión previenen trabajos no recogidos y optimizan el flujo de trabajo en grandes redes empresariales.

Server Message Block (SMB)

Redes Windows con SMB

Características del Protocolo SMB

  • Puerto: TCP/445 directamente
  • Versión actual: SMB3
  • Compatibilidad UNIX/Linux: Suite Samba

Puertos SMB

TCP/445: SMB directo sobre TCP

TCP/139: SMB sobre NetBIOS (legacy)

⚠️ Alerta de Seguridad

SMB1 está deshabilitado por defecto en versiones actuales de Windows debido a vulnerabilidades críticas de seguridad. Solo SMB2/3 deben utilizarse en entornos modernos.

CIFS (Common Internet File System): Técnicamente se refiere solo a un dialecto específico de SMB versión 1, aunque a menudo se usa incorrectamente como sinónimo de SMB.

Network Basic Input/Output System (NetBIOS)

NetBIOS y NetBT

  • NetBIOS original: Stack de protocolos pre-TCP/IP
  • NetBT: NetBIOS over TCP/IP
  • Función: Direccionamiento por nombre y establecimiento de sesiones

Puertos NetBT

UDP/137: Servicios de nombres

UDP/138: Conexiones UDP

TCP/139: Servicios de sesión TCP

Sistemas legacy de red

🚨 Recomendación de Seguridad

NetBT debe deshabilitarse en la mayoría de redes modernas ya que representa un riesgo significativo de seguridad. Solo es necesario para soporte de versiones de Windows anteriores al 2000.

Modernización: Las redes modernas utilizan IP, TCP/UDP y DNS para estas funciones, haciendo NetBT obsoleto.

File Transfer Protocol (FTP)

Transferencia de archivos por red

Características de FTP

  • Función: Subida y descarga de archivos
  • Uso común: Subida de archivos a sitios web
  • Puerto de control: TCP/21
  • Puerto de datos: TCP/20 (activo) o asignado por servidor (pasivo)

Modos de Funcionamiento

  • Modo Activo: Servidor inicia conexión de datos
  • Modo Pasivo: Cliente inicia ambas conexiones

⚠️ Riesgo de Seguridad

FTP plano no está cifrado y representa un alto riesgo de seguridad. Las contraseñas se envían en texto plano.

FTPS

FTP-Secure con cifrado SSL/TLS

SFTP

Servidores Web

Infraestructura de servidores web

Protocolo HTTP

  • Puerto por defecto: TCP/80
  • Función: Permitir a clientes (navegadores) solicitar recursos
  • Método principal: GET para obtener datos
  • Respuestas: Datos solicitados o códigos de error

Capacidades Extendidas

  • HTML: Páginas con formato y hiperenlaces
  • POST: Envío de datos del cliente al servidor
  • Scripting: Aplicaciones web programables

Tipos de Implementación

  • Internet Público: Sitios web accesibles globalmente
  • Intranet: Redes privadas solo con acceso local
  • Extranet: Redes privadas con acceso remoto autorizado

Uniform Resource Locators (URLs)

Componentes de una URL

https://store.comptia.org/bundles/aplus.html

Elementos principales

  • Protocolo: https:// (método de acceso)
  • FQDN: store.comptia.org (ubicación del host)
  • Ruta: /bundles/aplus.html (directorio y archivo)

Consideraciones:

  • FQDN no es sensible a mayúsculas
  • La ruta puede ser sensible a mayúsculas
  • Se puede usar dirección IP en lugar de FQDN
  • IPv6 debe ir entre corchetes

Navegador web mostrando URL

HyperText Transfer Protocol Secure (HTTPS)

Navegador con conexión segura HTTPS

Seguridad en HTTP

  • Problema HTTP: Datos no cifrados, sin autenticación
  • Solución: SSL/TLS desarrollado por Netscape
  • Puerto HTTPS: TCP/443
  • Estándar actual: TLS (Transport Layer Security)

Otros protocolos seguros

  • FTPS, SMTPS, LDAPS
  • POP3S, IMAPS
  • DTLS para UDP (VPNs)

Implementación de Certificados

  • Certificado digital: Emitido por Autoridad Certificadora (CA)
  • Par de claves: Privada (secreta) y pública (en certificado)
  • Túnel cifrado: Comunicaciones protegidas
  • Indicador visual: Candado en barra de direcciones

Servidores de Correo Electrónico

Tipos de Protocolos de Correo

  • Transferencia de correo: SMTP para entrega
  • Acceso a buzón: POP3/IMAP para descarga

Formato de Direcciones

usuario@dominio.com david.martin@comptia.org

Proceso de Entrega

  1. Cliente envía a servidor SMTP local
  2. SMTP busca MX record en DNS
  3. Establecimiento de sesión con servidor remoto
  4. Entrega a buzón del destinatario

Infraestructura de correo electrónico

Simple Mail Transfer Protocol (SMTP)

Transferencia de correo entre servidores

Configuraciones de Puerto SMTP

Puerto TCP/25

  • Retransmisión entre servidores SMTP
  • Message Transfer Agents (MTAs)
  • Inseguro: Sin cifrado

Puerto TCP/587

  • Envío desde clientes de correo
  • Message Submission Agents (MSAs)
  • Seguro: Con cifrado y autenticación

Descubrimiento de servidores: SMTP utiliza registros MX y A/AAAA en DNS para localizar servidores de correo del dominio de destino.

Recomendación: Siempre configurar puerto 587 con cifrado TLS y autenticación para clientes de correo.

Post Office Protocol 3 (POP3)

Características de POP3

  • Puerto inseguro: TCP/110
  • Puerto seguro: TCP/995 (POP3S)
  • Funcionamiento: Descarga y eliminación
  • Autenticación: Usuario y contraseña

Limitaciones

  • Acceso desde un solo dispositivo
  • Mensajes eliminados del servidor
  • Sin sincronización entre dispositivos
  • Gestión limitada de carpetas

Opción: Algunos clientes permiten dejar mensajes en el servidor para acceso múltiple.

Cliente de correo configurando POP3

Internet Message Access Protocol (IMAP)

IMAP permite acceso multi-dispositivo

Ventajas de IMAP sobre POP3

  • Puerto inseguro: TCP/143
  • Puerto seguro: TCP/993 (IMAPS)
  • Conexiones permanentes: Al servidor
  • Múltiples clientes: Acceso simultáneo

Características Avanzadas

  • Gestión de buzón en el servidor
  • Organización en carpetas
  • Control de eliminación de mensajes
  • Múltiples buzones por usuario
  • Sincronización entre dispositivos

Ideal para: Usuarios que acceden al correo desde múltiples dispositivos (PC, tablet, smartphone) y necesitan sincronización completa.

Servidores de Directorio y Autenticación

Necesidad de Control de Acceso

  • Autenticación: Verificar identidad de usuarios
  • Autorización: Controlar acceso a recursos
  • Single Sign-On (SSO): Una autenticación para todos los servicios

Evolución de Sistemas

  • Workgroup: Contraseña compartida simple
  • Enterprise: Base de datos centralizada de cuentas
  • Directorio: Servicios compatibles con aplicaciones

Servicios de directorio empresarial

Beneficio clave: Los usuarios se autentican una vez y obtienen acceso autorizado a todos los servidores de aplicaciones compatibles en la red.

Lightweight Directory Access Protocol (LDAP)

Estructura de directorio LDAP

Características LDAP

  • Estándar base: X.500
  • Puerto inseguro: TCP/UDP 389
  • Puerto seguro: TCP/636 (LDAPS)
  • Función: Consultar y actualizar directorios

Implementaciones Populares

  • Windows Active Directory: Microsoft
  • OpenLDAP: Código abierto
  • Otros: Apache Directory, 389 Directory

Estructura de Directorio

Los recursos de red se registran como objetos dentro del directorio, donde cada objeto es como un registro y los atributos son como campos de base de datos.

Authentication, Authorization, and Accounting (AAA)

Componentes AAA

  • Supplicant: Dispositivo solicitando acceso (PC, laptop)
  • NAS/NAP: Dispositivos de acceso a red (switches, APs, VPN gateways)
  • AAA Server: Servidor de autenticación en la red local

Protocolos AAA

RADIUS

TCP/UDP 1812/1813
Autenticación de usuarios

TACACS+

TCP/49
Autenticación de dispositivos

Arquitectura AAA con RADIUS

Ventaja: Los dispositivos de acceso no almacenan credenciales, solo actúan como tránsito entre el servidor AAA y el suplicante.

Servidores de Acceso Terminal Remoto

Acceso remoto a terminal

Conceptos Fundamentales

  • Terminal: Punto final de comunicación (TTY)
  • Shell: Entorno de procesamiento de comandos
  • Emulador de terminal: Software que replica función TTY

Protocolos de Acceso

SSH (Secure Shell)

Puerto: TCP/22

Seguridad: Cifrado completo

Telnet

Puerto: TCP/23

Seguridad: Sin cifrado (obsoleto)

⚠️ Recomendación de Seguridad

Solo usar SSH para acceso remoto. Telnet transmite credenciales en texto plano y es vulnerable a packet sniffing.

Secure Shell (SSH)

Características de SSH

  • Puerto: TCP/22 por defecto
  • Cifrado: Toda la comunicación protegida
  • Plataformas: UNIX, Linux, Windows, macOS
  • Implementación popular: OpenSSH

Usos Adicionales

  • SFTP: Transferencia segura de archivos
  • Túneles: Port forwarding
  • Gestión de red: Switches, routers, firewalls
  • Automatización: Scripts y administración

Terminal SSH con cifrado

SSH es el estándar de facto para acceso seguro remoto a servidores UNIX/Linux y dispositivos de red empresariales.

Remote Desktop Protocol (RDP)

Conexión RDP a escritorio Windows

Características RDP

  • Puerto: TCP/3389
  • Desarrollador: Microsoft
  • Función: Acceso GUI remoto a Windows
  • Cifrado: Configurable por administrador

Compatibilidad Multi-plataforma

  • Clientes: Linux, macOS, iOS, Android
  • Servidores: xrdp para sistemas no-Windows
  • Permisos: Configurables por administrador

Ventaja: Permite administración gráfica completa, ideal para usuarios que prefieren interfaces visuales sobre línea de comandos.

Servidores de Tiempo

Network Time Protocol (NTP)

  • Puerto: UDP/123
  • Función: Sincronización de relojes en red
  • Importancia: Logs precisos y protocolos temporizados

Niveles de Stratum

  • Stratum-0: Reloj atómico (fuente más precisa)
  • Stratum-1: Servidores NTP conectados a Stratum-0
  • Stratum-2: Servidores sincronizados con Stratum-1
  • Stratum-n: Niveles subsecuentes

Red de servidores NTP

# Verificar estado NTP en Windows w32tm /query /status

Network Time Security (NTS)

Aprobado en 2020: Usa cifrado TLS en puerto TCP/4460 para proteger la transferencia de datos de tiempo.

Windows por defecto: Utiliza time.windows.com como servidor de tiempo.

Simple Network Management Protocol (SNMP)

Sistema de monitoreo SNMP

Componentes SNMP

  • Sistema de gestión: Estación de monitoreo
  • Dispositivos gestionados: Switches, routers, servidores
  • Agentes: Proceso en cada dispositivo

Puertos SNMP

UDP/161: Consultas a dispositivos

UDP/162: Traps (alertas)

Versiones

  • SNMPv1/v2: Inseguros, no usar
  • SNMPv3: Con autenticación, recomendado

MIB (Management Information Base): Base de datos en cada agente que contiene estadísticas sobre la actividad del dispositivo.

Syslog - Recolección de Logs

Características Syslog

  • Puerto: UDP/514
  • Función: Recolector centralizado de logs
  • Estándar: De facto para logging distribuido
  • Compatibilidad: Routers, switches, servidores UNIX/Linux

Formato de Mensaje Syslog

  • PRI code: Calculado de facility y severity
  • Header: Timestamp y hostname
  • Message: Tag del proceso y contenido

Sistema centralizado de logs

Beneficios de Centralización

  • Revisión eficiente de logs
  • Respuesta rápida a alertas
  • Dashboards de estado
  • Correlación de eventos

Servidores Proxy

Proxy como intermediario web

Diferencias con NAT

  • NAT: Solo traduce direcciones IP
  • Proxy: Procesa requests HTTP completos
  • Puerto común: 8080 por convención

Tipos de Proxy

  • Transparente: Sin configuración cliente
  • No transparente: Requiere configuración

Funciones Adicionales

  • Filtrado de contenido: Bloqueo de sitios
  • Cache: Mejora rendimiento
  • Control de acceso: Reglas y horarios

Dispositivos de Seguridad y UTM

Funciones de Seguridad

  • Firewalls: Control de acceso por ACL
  • IDS/IPS: Detección/prevención de intrusiones
  • Antivirus: Escaneo de archivos en tránsito
  • Spam Gateway: Filtrado de correo usando SPF/DKIM/DMARC
  • Content Filter: Bloqueo de sitios web
  • DLP: Prevención de fuga de datos

Unified Threat Management (UTM)

  • Combina múltiples funciones de seguridad
  • Configuración y reporting centralizados
  • Alternativa a dispositivos separados

Dispositivo UTM empresarial

Ventaja UTM: Simplifica la gestión de seguridad al centralizar múltiples funciones en un solo dispositivo con configuración y reportes unificados.
